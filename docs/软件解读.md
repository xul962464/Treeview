# 软件解读

这份文档用于快速定位当前程序的主要功能和对应代码入口，方便后续继续迭代时减少重复摸索。

## 1. 总体结构

- 启动入口: `app/main.py`
  - 创建 `QApplication`
  - 创建并显示 `MainWindow`
- 主窗口: `app/gui/main_window.py`
  - 左侧菜单和所有业务操作入口
  - 文件导入、导出、撤销重做、批量调整、分组管理
  - 负责把界面状态同步到 `TreeView`
- 视图渲染: `app/gui/tree_view.py`
  - 负责树的图形绘制
  - 负责矩形树、环形树、副图、比例尺、分组标注、节点标签
  - 负责场景选择、双击编辑、拖动、滚轮缩放
- 注释状态: `app/gui/annotation_state.py`
  - 保存样品标签覆盖、节点标签覆盖、比例尺标签覆盖、分组、底色、偏移等
- 树模型: `app/phylo/model.py`
  - `TreeModel`
  - `TreeNode`
- 文件解析: `app/phylo/parse.py`
  - 负责 `.nwk/.newick/.tre/.nex/.nexus`
  - 当前支持标准 Newick、标准 NEXUS、带 `translate` 和 BEAST 注释的 NEXUS 回退解析
- 样式配置: `app/phylo/style_config.py`
  - 配置文件导入后应用的标签样式和注释

## 2. 主要数据流

### 2.1 打开树文件

- 入口: `MainWindow._open_tree_file`
- 加载: `MainWindow._load_tree_into_viewer`
- 解析: `load_trees()` in `app/phylo/parse.py`
- 成功后:
  - 重置 `AnnotationState`
  - 重置历史栈
  - 调用 `_rerender_current_tree(auto_fit=True)`

### 2.2 渲染树

- 入口: `MainWindow._rerender_current_tree`
- 将当前模型和注释状态应用给 `TreeView`
- `TreeView.render_tree(...)`
  - `rectangular`: `_render_rectangular`
  - `circular`: `_render_circular`
  - 若启用副图: `_render_inset_overview`

### 2.3 撤销重做

- 状态快照: `HistoryState` in `main_window.py`
- 捕获: `_capture_history_state`
- 撤销/重做: `_undo` / `_redo`
- 当前已优化:
  - 视图参数未变化时不再压历史
  - 拖动类视图操作只有真的发生变化才压历史

## 3. 功能定位速查

### 3.1 文件导入导出

- 打开文件: `MainWindow._open_tree_file`
- 导出 PNG/PDF/NWK/状态: `MainWindow._export_*`
- 一键导出: `MainWindow._export_bundle`

### 3.2 树解析兼容

- 主入口: `app/phylo/parse.py -> load_trees`
- `.tre` 内容判断:
  - 若看起来像 NEXUS，优先按 NEXUS 解析
  - 否则按普通 Newick 解析
- BEAST/NEXUS 回退:
  - `_parse_nexus_fallback`
  - `_normalize_beast_newick`
  - `_parse_translate_block`
- 支持度提取:
  - `_parse_support`
  - `_extract_support_value`

### 3.3 样品名与富文本

- 双击编辑文本: `MovableTextItem` in `tree_view.py`
- 样品标签绘制: `_draw_tip_label_rect`, `_draw_tip_label_circular`
- 样品标签格式应用:
  - `MainWindow._edit_selected_tip_font`
  - `TreeView._create_tip_label_item`

### 3.4 节点标签与置信度

- 节点标签绘制:
  - 矩形树: `_render_rectangular` 中的 `show_support_labels`
  - 环形树: `_render_circular` 中的 `show_support_labels`
- 节点标签 item: `_create_node_label_item`
- 默认置信度文本:
  - `TreeView._format_support_text`
  - `MainWindow._node_default_label_text`
- 置信度显示格式:
  - `TreeRenderOptions.support_display_format`
  - `MainWindow._cmb_support_display`

### 3.5 比例尺

- 主比例尺绘制: `TreeView._draw_scale_bar`
- 副图比例尺绘制: `TreeView._draw_inset_scale_bar`
- 比例尺文字编辑:
  - `MainWindow._edit_scale_bar_text`
  - `TreeView._create_scale_bar_item`

### 3.6 副图

- 开关和位置/大小参数:
  - `TreeRenderOptions.inset_overview_*`
- 副图绘制:
  - `TreeView._render_inset_overview`
- 副图拖动/缩放信号:
  - `TreeView.insetOverviewMoved`
  - `TreeView.insetOverviewScaleChanged`
- 主窗口同步:
  - `_on_inset_overview_moved`
  - `_on_inset_overview_scale_changed`

### 3.7 分组与底色

- 新增分组/底色:
  - `_create_group_from_selection`
  - `_create_background_from_selection`
  - `_append_leaf_group`
- 列表刷新: `_refresh_group_list`
- 改名/改色/删除:
  - `_rename_group`
  - `_recolor_group`
  - `_delete_group` / `_delete_groups`
- 渲染:
  - 矩形背景: `_draw_group_backgrounds_rectangular`
  - 环形背景: `_draw_group_backgrounds_circular`
  - 矩形分组标注: `_draw_groups_rectangular`
  - 环形分组标注: `_draw_groups_circular`

### 3.8 自动调整与排序

- 智能排序: `_sort_tree_by_topology_depth`
- 自动调整: `_auto_adjust_tree`
- 名称快速替换:
  - `_quick_replace_accession_format`
  - `_quick_replace_underscore_format`
- 一键斜体:
  - `_italicize_all_tip_labels`

## 4. 当前实现上的关键约束

- `TreeView.render_tree()` 每次都会清空并重建场景 item
  - 好处: 逻辑简单
  - 代价: 大树下重绘成本较高
- 主树和副图是同一个 scene，但副图不进入主树的 item 映射
  - 这样不会污染主树选择、文本编辑、右键菜单
- 分组仍要求叶序连续
- 撤销重做仍基于状态快照，不是增量 patch

## 5. 后续继续改时的推荐顺序

如果下次继续改功能，通常建议先定位：

1. 文件解析问题:
   - `app/phylo/parse.py`
2. 菜单、按钮、交互入口:
   - `app/gui/main_window.py`
3. 实际绘图、坐标、颜色、文本:
   - `app/gui/tree_view.py`
4. 保存/恢复/撤销:
   - `HistoryState`
   - `_serialize_annotation_state`
   - `_deserialize_annotation_state`
   - `_load_tree_state`
   - `_tree_state_payload`

## 6. 快速排错提示

- “文件无法导入”:
  - 先看 `parse.py -> load_trees`
- “显示出来了但标签/比例尺不对”:
  - 先看 `tree_view.py`
- “界面参数改了但主视图没变”:
  - 先看 `main_window.py -> _mutate_view_options`
- “撤销后状态不对”:
  - 先看 `HistoryState` 和 `_capture_history_state / _apply_history_state`
- “导入状态文件后不一致”:
  - 先看 `_tree_state_payload` / `_load_tree_state`
